拥抱 Cursor

起因是 deepseeek 过于火爆，以至于经常不可用，网页端总是【服务器繁忙，请稍后重试】，L 站上有个有趣的帖子就是在讨论这个现象。字节前段时间发布的 trae 免费，却总是发生登录验证问题，某天一直登录失败，放弃使用。虽然现在 trae 似乎又好了。

一直不使用 cursor 的原因是免费额度优先，暂时没有那么大使用 AI 编程的需求，且有不少替代方案可以使用。正好这次能用来编程的备用方案都不可用了，就试试 Cursor 吧。

使用 50 次免费额度给博客增加了热力图功能后，我觉得 Cursor 的可用性比之前使用的 Roo Code 使用 deepseek-v3 api 的能力强上许多。最开始尝试使用 Roo Code 也是看到 B 站有 Up 主发视频说可以平替 Cursor，那次也是我第一次尝试使用 AI 描述性编程，之前最多是 Copilot 模式的辅助编程。第一次尝试总会受到震惊，大力夸赞Roo Code 和 deepseek-v3 的能力。不过从现在的角度再去评价一下这个编码组合，能力还是不够。

这周高强度了解了下 Cursor 的相关信息，把官网扫了一遍。把我觉得主要有用的信息罗列在下面：

  • 设置页面的模型描述和用量记录。

    • GPT-4, GPT-4o, and Claude 3.5 Sonnet (Claude 3.5 Haiku is 1/3 of a premium request). You may use unlimited slow requests (may run slower during high demand).

    • 其他类型的模型不限量，但是仍然会记录使用的各类模型记录，点击 Enable usage-based pricing 就可以看到使用模型的历史记录。不过要记得关掉，不然开启后会在优先额度用完后，继续额外付费使用优先额度。
    • 题外话，2 月 15 日，o3-mini 模型降价到了 1/3 优先额度。Update: o3-mini pricing
  • 官方文档，描述了 Cursor 的主要功能和一些使用提示。
    • Tab 是手动编码过程中 Cursor 使用最多的功能。
    • Chat 和 Composer 是指挥 AI 编码过程中使用最多的功能。
      • Chat 主要用于梳理需求，和 AI 聊天，获取信息，理清思路。
      • Composer 基于 Agent，在描述需求的基础上，自动编码。
    • Git commit 信息总结是我用得非常开心的一个功能。
    • 各类 @ 功能，用于更清楚的描述问题。
    • cursor rules 和 cursor ignore 定制项目偏好。
    • cmd/ctrl + k，在编辑器的各个地方和 AI 对话。
  • 模型页面，设置里的模型页面也是了解各个模型价格的宁一个来源。

Cursor 的上手还是比较符直觉的，即使不怎么主动去了解 Cursor 的一些使用技巧，也可以用好 Cursor 的强大能力。符合只觉得产品才是好的产品。

个人觉得如果编程的需求不多，Cursor 免费额度，Trae 如果登录没有问题，可以免费使用，Roo Code 使用 deepseek v3 api，基本已经足够满足需求，以上这些产品本人都用过，不如 Cursor，但是足够使用。

我已经付费 20 刀使用 Cursor 了，就不折腾使用这些替代品了，接下来一个月左右，好好体验下。不过大模型高速发展的时代，按月付费就好，下个月也许就有更好的产品了。

2 月 16 下午更新：Cursor 已经给我的付费账户降智了，现在就看一个月下来降智的频率，如果比较多的话，不再考虑付费了。在官方论坛我的两个反馈：1、2，就是不知道官方是否会回复。

AI 编辑器的价格

虽然 20 刀购买了 Cursor 一个月的使用权，也有关注其他编辑器的价格。GitHub Copilot 10 刀/月，Windsurf 老用户 10 刀/月，新用户 15 刀/月，都比 Cursor 便宜一些，不过能力上都欠缺一些。如果这个月我的 Cursor 额度没用完，我就还是购买便宜点试试。

域名邮箱

借助网上的教程，用 Cloudflare + Resend + Gmail + 域名，给自己整了个 mail@guanglai.me 的域名邮箱。每天可以发最多 100 封邮件，收邮件是不记额度的。

Cloudflare 用来把收到的邮件转发到我的 gmail 邮箱。resend 提供域名邮箱的发送能力，可使用 gmail 邮箱对外转发。
